# 覆写 App · 技术栈设计

**日期**：2026-04-08  
**范围**：技术栈选型与整体架构

---

## 背景

「覆写」是一款基于 7 个心理机制的 365 天每日行为练习 App，仅供个人使用（iOS）。

**约束条件**：
- 开发环境：Windows，无 Mac
- 平台：iOS 仅限个人使用，通过 URL 访问并添加到主屏幕
- 开发经验：零基础
- 通知：每日提醒是核心功能，不可缺失

---

## 技术栈选型

**方案**：PWA（渐进式 Web 应用）

**选型理由**：
- Windows 开发者无法本地编译 iOS App（无 Xcode）
- 个人使用无需上架 App Store，无需 Apple Developer 账号（¥700/年）
- iOS 16.4+ 已支持 Web Push，满足通知需求
- 迭代效率高：推送代码即发布，无需等待编译

**前提条件**：用户 iPhone 为 iOS 16.4 或以上。

---

## 整体架构

```
覆写 PWA
├── 前端：React + Vite + Tailwind CSS
│
├── 数据层（全本地）
│   ├── exercises.json      ← 基础练习库（60个，静态打包，不变）
│   └── localStorage
│       ├── mechanism_progress  ← 7个机制的难度水位和连续天数
│       ├── exercise_log        ← 每日完成记录和感受文字
│       ├── daily_plan_{date}   ← 当天3个练习及完成状态
│       ├── dynamic_exercises   ← Claude 每周生成的个性化练习
│       └── settings            ← 通知时间、push订阅信息、开始日期
│
├── 后端（Vercel Serverless Functions）
│   ├── /api/push-notify    ← Cron 每天 8:00 触发，发 Web Push 通知
│   └── /api/weekly-review  ← 接收本周数据，调用 Claude API，返回结果
│
├── AI 层：Claude API（通过后端中转，key 不暴露前端）
│
└── 部署：Vercel（免费额度）
    └── GitHub push → 自动发布
```

### 练习库合并规则

推送每日练习时，从两个来源合并选取：
1. `exercises.json`（基础库，60个）
2. `localStorage.dynamic_exercises`（Claude 生成库）

相同 `id` 时，动态库优先（Claude 版本覆盖基础版本）。

## 通知层设计

**技术**：Web Push API + Service Worker + VAPID

**流程**：
1. 用户首次打开 App → 授权通知 → 浏览器生成 Push Subscription
2. Subscription 存入 `localStorage.settings.push_subscription`
3. 用户触发「注册通知」→ 前端将 Subscription 发送至 `/api/push-notify` 存储
4. Vercel Cron 每天 8:00 执行 → 调用 `/api/push-notify` → 向用户设备发送 Web Push

**服务端订阅存储**：个人单设备使用，Push Subscription 存为 Vercel 环境变量（`PUSH_SUBSCRIPTION`），注册时由前端调用 `/api/push-notify` 写入，无需数据库。

**注意**：用户必须将 PWA 添加到 iPhone 主屏幕，Web Push 才能在后台生效。Cron 时区设为 Asia/Shanghai，8:00 CST 触发。

## AI 层设计

### 触发时机

每周（默认周日晚）用户手动点击「查看本周回顾」触发。

### 数据流

```
前端                    后端(/api/weekly-review)          Claude API
  │                             │                              │
  │── POST 本周数据 ──────────>│                              │
  │   {                         │                              │
  │     week_logs,              │── 构造 prompt ────────────>│
  │     mechanism_progress,     │                              │
  │     start_date              │<─ 返回结构化 JSON ──────────│
  │   }                         │                              │
  │<── 返回结果 ────────────────│                              │
  │   {                         
  │     report_text,            ← 周报文字（直接展示）
  │     water_level_adjustments,← 各机制水位增减值
  │     new_exercises            ← 新增/修改的练习（JSON）
  │   }
```

### Claude 输出格式

```json
{
  "report_text": "这周你在M1上有明显的突破...",
  "water_level_adjustments": {
    "M1": +5,
    "M2": -3,
    "M3": 0
  },
  "new_exercises": [
    {
      "id": "CLAUDE-002",
      "mechanism": "M2",
      "difficulty": 45,
      "title": "...",
      "instruction": "...",
      "why": "...",
      "completion_prompt": "...",
      "simplified_versions": []
    }
  ]
}
```

## 开发顺序建议

1. **P0**：静态练习库 + 每日3练习展示 + 完成/跳过交互 + 感受记录
2. **P1**：难度水位系统 + 降级机制 + 7机制进度页 + 日历热力图
3. **P2**：Web Push 通知 + 每周回顾 + Claude API 集成（动态练习库）
4. **P3**：Figma 设计稿落地（视觉精修）+ 数据导出
