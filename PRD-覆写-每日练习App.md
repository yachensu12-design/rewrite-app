# 手机 App 需求文档：「覆写」

## 一、产品概述

### 产品名称
覆写（Rewrite）

### 一句话描述
一款基于个人心理机制定制的每日行为练习 App，通过 365 天的渐进式微练习，帮助用户从旧的创伤剧本中一点一点长出新的行为模式。

### 核心理念
这个 App 不是心理咨询工具，不是冥想 App，不是鸡汤推送器。它的本质是一个**行为训练系统**——每天给用户三个具体的、可执行的小动作，这些动作直接对应用户已识别的七个潜意识机制。通过持续一年的重复练习，让身体积累足够多的"新体验"，从而慢慢覆写旧的自动反应。

### 核心受众
仅限个人使用。所有练习内容完全基于用户本人的心理机制定制，不需要个性化系统或问卷。

### 设计调性
- **温暖但不软弱**：像一个了解你的朋友，语气平静、不评判、不催促
- **克制**：不用感叹号、不用"加油"、不用"你很棒"
- **尊重用户的节奏**：做不到的时候不施压，系统会自动降级难度
- **安全感**：整体视觉让人放松，不像任务管理器，更像一本私人日记

## 二、七个心理机制（练习内容的底层框架）

所有练习围绕以下七个已识别的潜意识机制设计。每个机制内部独立分级，从最简单到最难渐进。七个机制同时进行，每天的三个练习从不同机制中抽取。

| 编号 | 机制名称 | 核心模式 | 练习方向 |
|------|----------|----------|----------|
| M1 | 自我消除 | 压抑需求以避免被惩罚 | 练习表达真实需求、不缩小自己 |
| M2 | 情绪容器 | 习惯性照顾别人情绪而忽略自己 | 练习觉察"我在照顾谁的情绪"、允许自己的感受优先 |
| M3 | 愤怒内折 | 对外的愤怒转向攻击自己 | 练习向外归因、允许愤怒存在 |
| M4 | 价值外包 | 用外部指标（金钱、关注）衡量自我价值 | 练习从内在体验而非外部数字确认自己 |
| M5 | 幻想系统 | 用幻想替代真实的连接 | 练习区分幻想和现实、待在当下 |
| M6 | 信任有效期 | 先理想化再等待幻灭 | 练习看到真实的人、不急于理想化或否定 |
| M7 | 将就模板 | 宁愿留在坏的关系里也不面对断裂 | 练习忍受断裂的不适、为自己争取 |

## 三、练习难度体系

### 3.1 难度不分固定等级，由系统动态评估

每个练习有一个内部难度分值（1-100），但不向用户展示具体数字。系统根据以下信号动态调整推送给用户的练习难度：

**升级信号**（推送更难的练习）：
- 连续 3 天完成某机制的练习
- 用户在感受记录中写了正面体验（如"比想象中简单"、"做到了"）
- 某机制的完成率超过 80%

**降级信号**（推送更简单的练习）：
- 某练习未完成，自动降级为该练习的简化版本
- 连续 2 天未完成某机制的练习
- 用户在感受记录中表达困难（如"做不到"、"太难了"）

**降级规则**：
- 每个练习有 1-3 个简化版本（见练习库设计）
- 降级时推送同一机制的更简单版本，而非跳过
- 最低级别的练习是纯觉察类（"今天注意一下你有没有出现XX反应"），不要求任何行动

### 3.2 难度维度

练习的难度由以下维度决定：

| 维度 | 低难度 | 中难度 | 高难度 |
|------|--------|--------|--------|
| 对象 | 自己（内在觉察） | 陌生人/不重要的人 | 重要的人（朋友、母亲、伴侣） |
| 行动 | 想一想 | 感受身体反应 | 做一个具体动作 |
| 暴露程度 | 私密的（只有自己知道） | 半公开的（可能被注意到） | 公开的（需要和人互动） |
| 情绪强度 | 低（日常场景） | 中（有轻微不适） | 高（触碰核心恐惧） |

## 四、练习库设计

### 4.1 数据结构

```json
{
  "exercises": [
    {
      "id": "M1-001",
      "mechanism": "M1",
      "difficulty": 15,
      "title": "注意「随便」",
      "instruction": "今天如果有人问你想吃什么、想去哪里、想做什么，注意你有没有说「随便」或「都行」。不需要改变它，只需要注意到。",
      "why": "你说「随便」的时候，往往不是真的没有偏好——而是你的身体在自动执行「不占空间」的程序。",
      "completion_prompt": "今天你注意到了吗？",
      "simplified_versions": [
        {
          "difficulty": 5,
          "instruction": "今天回忆一下，你最近一次说「随便」或「都行」是什么时候？当时你真正想要的是什么？"
        }
      ]
    }
  ]
}
```

### 4.2 字段说明

- `id`：唯一标识，格式为 `{机制编号}-{序号}`
- `mechanism`：所属机制（M1-M7）
- `difficulty`：内部难度分值（1-100），不向用户展示
- `title`：练习标题（简短，3-8个字）
- `instruction`：练习内容（具体、可执行、不超过 100 字）
- `why`：为什么做这个练习（一句话解释背后的心理机制，不超过 60 字）
- `completion_prompt`：完成后的引导语（引导用户写感受）
- `simplified_versions`：降级版本数组（1-3个，难度递减）

### 4.3 练习内容（首批，每个机制 8-10 个，共约 60 个）

#### M1 自我消除（练习表达需求、不缩小自己）

**M1-001 注意「随便」** (难度 10)
- 今天如果有人问你想吃什么、想去哪里、想做什么，注意你有没有说「随便」或「都行」。不需要改变它，只需要注意到。
- 简化版：回忆一下你最近一次说「随便」是什么时候。

**M1-002 说出一个偏好** (难度 25)
- 今天当有人问你意见时，说出你真正的偏好。不说「随便」，不说「都行」，说一个具体的选择。
- 简化版：在心里默默想出你的真实偏好，不需要说出口。

**M1-003 点你想吃的** (难度 20)
- 今天吃饭的时候（不管是外卖还是餐厅），点你真正想吃的东西。不点最便宜的，不点别人推荐的，点你自己最想要的。
- 简化版：打开外卖 App，浏览菜单，找到你最想吃的那个——不需要真的下单。

**M1-004 不解释你的选择** (难度 35)
- 今天当你做了一个选择（穿什么、吃什么、去哪里），如果有人问「为什么」，试着只说「因为我想」。不解释理由，不证明它是合理的。
- 简化版：注意你有没有在做选择后自动开始解释的冲动。

**M1-005 说「我不想」** (难度 40)
- 今天如果有人邀请你做一件你不想做的事，说「我不太想」。不需要给理由，不需要道歉。
- 简化版：在心里对那件你不想做的事说「我不想」。感受说出这三个字时身体的反应。

**M1-006 不压下不满** (难度 55)
- 今天如果有一个让你不舒服的瞬间（哪怕很小），试着不压下去。可以皱眉、可以叹气、可以说「这让我有点不舒服」。
- 简化版：今天注意你有几次压下了不满。数一数就好。

**M1-007 主动提一个要求** (难度 65)
- 今天向一个人主动提一个小要求。比如「你能帮我带杯水吗」「你能把这个发给我吗」。不是你不能自己做——而是练习「我可以向别人提出需求」。
- 简化版：想一个你想向别人提但还没提过的要求。写下来。

**M1-008 告诉母亲你的真实感受** (难度 85)
- 下次你母亲否定你的感受时（比如「你怎么会累呢」），试着说一句：「我就是累了。」不解释，不争辩。就是一句平静的陈述。
- 简化版：在心里对她说「我就是累了」。感受说这句话时的身体反应。

#### M2 情绪容器（练习觉察自己在照顾谁的情绪）

**M2-001 今天我照顾了谁的情绪** (难度 10)
- 睡前回忆一下今天的互动。有没有哪个瞬间，你压下了自己的感受，去照顾了别人的情绪？不需要改变，只需要看到。

**M2-002 不主动安慰** (难度 30)
- 今天如果有人在你面前抱怨或诉苦，试着不立刻安慰、不立刻给建议。就听着。注意你体内那个想要「帮忙」的冲动。
- 简化版：注意你在别人诉苦时，身体的第一反应是什么。

**M2-003 先问自己** (难度 35)
- 今天在回应别人之前，先在心里问一句：「我现在自己的感受是什么？」然后再决定要不要回应对方。

**M2-004 不回母亲的消息（实验）** (难度 60)
- 今天如果你母亲给你发了不紧急的消息，试着延迟回复——不是不回，是晚几个小时回。注意你在这几个小时里的身体感受。愧疚感有多强？
- 简化版：想象一下你不回她消息的场景。你的身体有什么反应？

**M2-005 说「我今天自己也不太好」** (难度 50)
- 今天如果有人需要你的情绪支持，但你自己也不好的时候，试着说：「我今天自己也不太好，可能没法很好地陪你。」
- 简化版：在心里默念这句话。感受说出来的阻力有多大。

#### M3 愤怒内折（练习允许愤怒向外）

**M3-001 注意自我攻击** (难度 10)
- 今天注意你有没有在某个瞬间对自己说「我怎么又这样」「我真差劲」「都是我的问题」。数一数有几次。

**M3-002 换一个归因** (难度 25)
- 今天如果你发现自己在自我攻击，试着问一句：「如果这件事发生在我最好的朋友身上，我会觉得是她的问题吗？」
- 简化版：回忆最近一次自我攻击的瞬间，用朋友的视角重新看一遍。

**M3-003 允许不满停留** (难度 40)
- 今天如果有人做了让你不舒服的事，在你的第一反应（自我攻击）出现后，试着让另一个声音也出现：「也许这不是我的问题。」让两个声音并存 10 秒钟。

**M3-004 写下你的愤怒** (难度 35)
- 打开备忘录，写下一件让你愤怒的事。不需要理性，不需要分析，不需要公平。就是你生气了，写下来。写完不需要做任何事。

**M3-005 对一件小事说「不行」** (难度 55)
- 今天对一件让你不满的小事（服务员上错菜、快递放错位置、同事推给你的活）直接说出来。不需要发火，平静地说「这个不对」就够了。
- 简化版：在心里说「这个不对」。

#### M4 价值外包（练习从内在体验确认自己）

**M4-001 看一眼银行余额** (难度 15)
- 打开银行 App，看一眼余额。不分析，不评判。看完之后注意你身体的感受——紧缩？焦虑？还是比想象中平静？
- 简化版：想象一下你打开银行 App 看到余额的场景。你的身体有什么反应？

**M4-002 不算他花了多少** (难度 30)
- 今天如果有人请你吃饭或为你花钱，试着不在心里默默计算金额。把注意力放在「我和他在一起的感觉怎么样」而不是「他花了多少」。

**M4-003 用自己的钱善待自己** (难度 25)
- 今天用自己的钱买一样让你开心的东西。买的时候在心里说：「这是我自己挣的钱，我在用它照顾我自己。」
- 简化版：想一样你想为自己买但一直没买的东西。写下来。

**M4-004 不比较** (难度 35)
- 今天如果你发现自己在和别人比较（他有的我没有、她的生活比我好），停下来 3 秒钟，对自己说：「她的拥有不会减少我的价值。」

**M4-005 为自己争取一笔钱** (难度 70)
- 处理一件你一直「嫌麻烦」而没去做的和钱相关的事：申诉一笔不合理的扣费、要求退款、追回押金。哪怕只是发一封邮件。
- 简化版：把这件事的第一步写下来（比如找到客服电话号码）。

#### M5 幻想系统（练习待在当下）

**M5-001 标记幻想** (难度 10)
- 今天注意你有没有开始幻想一段关系——和某个人在一起的未来图景、他会怎么对你、你们的生活会是什么样。不需要打断它，只需要在心里说：「我在幻想了。」

**M5-002 回到身体** (难度 20)
- 当你发现自己陷入幻想或反复品味旧关系时，把注意力拉回你的身体——你坐在哪里、脚踩着什么、空气是什么温度。停留 30 秒。

**M5-003 描述他的真实行为** (难度 35)
- 如果你正在对某个人产生强烈的感觉，拿出手机在备忘录里写下 3 件他实际做过的事（不是你想象中的，是真实发生的）。看看这 3 件事和你脑子里的故事之间的差距有多大。

**M5-004 不回味** (难度 45)
- 今天如果你发现自己在回味一段旧关系（重播对话、品味心碎），试着在 5 分钟内把注意力转移到一件需要动手的事情上——洗碗、整理桌面、走路。不是逃避，是练习不喂养幻想引擎。

**M5-005 一整天不查看他的社交媒体** (难度 55)
- 如果你正在对某个人上头，今天一整天不打开他的社交媒体主页。注意你想去看的冲动出现了几次。每次出现时，在心里说：「那是幻想在找燃料。」

#### M6 信任有效期（练习看到真实的人）

**M6-001 注意理想化** (难度 15)
- 今天注意你有没有把某个人抬到很高的位置——觉得他特别厉害、特别有魅力、特别与众不同。问自己：「这是我在看他，还是在看我为他写的剧本？」

**M6-002 列出一个人的三个普通特征** (难度 25)
- 想一个你正在理想化的人，列出他 3 个很普通的特征（比如他也会赖床、他的衣品其实一般、他说话有时候很无聊）。

**M6-003 不急于下结论** (难度 40)
- 今天如果有人做了一件让你失望的事，在你的翻译器把它翻译成「他果然不在意我」之前，试着想出 2 个其他的可能解释（他可能只是忙了、他可能没注意到）。

#### M7 将就模板（练习忍受断裂的不适、为自己争取）

**M7-001 注意「算了」** (难度 10)
- 今天注意你有没有在某个瞬间对自己说「算了」「就这样吧」「也行吧」。注意那个瞬间你真正的感受是什么——是真的无所谓，还是你在回避面对不满？

**M7-002 不降格** (难度 35)
- 今天如果你在某段关系或某件事里想要降低标准（「不恋爱也行」「能见面就行」「打折也行」），停下来问自己：「我真正想要的是什么？」把真正想要的写下来。

**M7-003 拒绝一个「将就」的选择** (难度 50)
- 今天拒绝一个你本来会「将就」接受的选择。可以是很小的事——换一个你不喜欢的菜、退回一件不合适的衣服、对一个不想去的聚会说「不去了」。

**M7-004 忍受一次「空」** (难度 65)
- 今天留出 10 分钟，什么都不做。不看手机、不听音乐、不分析自己。就坐着。注意那个「空」的感觉出现的时候，你的身体想要做什么（抓手机？找人聊天？开始分析？）。让自己在那个「空」里多待一会儿。
- 简化版：留出 3 分钟。

**M7-005 安静地不联系一个人** (难度 75)
- 如果你正在一段让你不舒服的关系里，今天不主动联系他。不是演戏，不是试探，就是不联系。注意你在不联系的过程中身体的反应。
- 简化版：推迟 2 小时再联系。

## 五、每日推送与调度算法

### 5.1 每日推送逻辑

每天早上（默认 8:00，可在设置中调整）一次性推送 3 个练习。

**抽取规则**：
1. 从 7 个机制中选 3 个不同的机制
2. 选择机制时参考以下权重：
   - 完成率最低的机制：权重 ×2（优先补弱项）
   - 连续完成 3 天以上的机制：权重 ×0.5（暂时减少，避免疲劳）
   - 昨天未完成的机制：权重 ×1.5（给第二次机会，但用降级版本）
3. 在选中的机制内，根据当前难度水位选择一个合适难度的练习
4. 三个练习按难度从低到高排列

### 5.2 难度水位系统

每个机制维护一个独立的「难度水位」（初始值 10，范围 1-100）：
- 完成一个练习：水位 +5
- 未完成一个练习：水位 -10
- 连续 3 天完成同一机制：水位 +10
- 连续 2 天未完成同一机制：水位 -15
- 用户感受中包含正面关键词：水位 +3
- 用户感受中包含困难关键词：水位 -5

推送练习时，选择难度分值最接近当前水位的练习。

### 5.3 降级机制

当一个练习未完成时：
1. 该练习第二天重新出现，但使用其 `simplified_versions` 中的第一个简化版本
2. 如果简化版本也未完成，使用下一级简化版本
3. 如果所有简化版本都未完成，该练习暂时搁置 7 天，改为推送同机制的其他更简单的练习

## 七、视觉设计规范

### 7.1 配色

| 用途 | 色值 | 说明 |
|------|------|------|
| 背景 | #FAF6F0 | 温暖米白 |
| 卡片背景 | #FFFFFF | 纯白 |
| 主文字 | #2E2720 | 深棕 |
| 辅助文字 | #8A7E72 | 中棕 |
| 淡化文字 | #B5A99A | 浅棕 |
| 强调色 | #B07A48 | 暖棕（用于按钮、进度条、重点） |
| 完成色 | #4A7A6A | 沉稳绿（用于完成状态） |
| 机制标识 | 七种淡色 | 每个机制一个辨识色 |

### 7.2 字体
- 标题：Noto Serif SC（衬线体，传达温暖和安静）
- 正文：Noto Sans SC 300/400（无衬线，清晰易读）
- 数字/标签：Noto Sans SC 500

### 7.3 圆角与阴影
- 卡片圆角：16px
- 按钮圆角：24px
- 卡片阴影：0 2px 12px rgba(0,0,0,0.04)

### 7.4 动画
- 完成练习：对勾从中心放大并淡入（300ms, ease-out）
- 卡片展开：高度动画 + 内容淡入（400ms）
- 周报/月报：逐段淡入（staggered, 200ms 间隔）
- 进度条：缓慢填充（600ms, ease-in-out）

## 八、技术方案建议

### 8.1 推荐技术栈
- **框架**：React Native（跨平台 iOS + Android）或 Flutter
- **状态管理**：本地存储为主（AsyncStorage / SharedPreferences）
- **数据库**：SQLite（本地存储所有练习记录和感受）
- **推送**：本地通知（不需要服务端推送）
- **数据格式**：JSON（练习库）+ SQLite（用户数据）

### 8.2 数据模型

```
users_progress
├── mechanism_id (M1-M7)
├── current_water_level (1-100)
├── total_completed
├── total_skipped
├── last_completed_date
└── consecutive_days

exercise_log
├── date
├── exercise_id
├── status (completed / skipped / auto_degraded)
├── feeling_text (用户写的感受)
├── difficulty_at_time
└── was_simplified (boolean)

daily_plan
├── date
├── exercise_1_id
├── exercise_2_id
├── exercise_3_id
└── completion_count
```
